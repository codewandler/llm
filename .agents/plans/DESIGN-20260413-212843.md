# Design: Unify Effort + Thinking Mode

**Date**: 2026-04-13
**Status**: Approved
**Scope**: `llm.Request` API, all providers, CLI

---

## Problem Statement

The current `llm.Request` has two overlapping fields for controlling model reasoning:

- `ThinkingEffort` (7 levels: none, minimal, low, medium, high, xhigh, unspecified) — tries to be both a **mode toggle** (on/off) and a **depth control** (how much)
- `OutputEffort` (5 levels: low, medium, high, max, unspecified) — Anthropic-specific band-aid that has no effect on any other provider

This causes several problems:

1. **`ThinkingEffort` does double duty.** "none" means "disable thinking" while "high" means "think a lot." These are different dimensions crammed into one field.
2. **`OutputEffort` is misnamed and leaks Anthropic internals.** It maps to Anthropic's `output_config.effort` which controls *all* token spend (not just output), and has no meaning on OpenAI or Bedrock.
3. **The promotion hack.** On Anthropic ≥ 4.6, `ThinkingEffort` must be promoted to `output_config.effort` because `budget_tokens` is rejected on adaptive thinking. This is fragile.
4. **Bedrock is out of sync.** It still has a hardcoded `switch` for budget mapping that doesn't use `ToBudget()`.
5. **OpenAI maps `ThinkingEffort` levels that don't exist in its API** (minimal, xhigh) with per-model-category translation tables.

---

## Design

Replace both fields with two orthogonal concepts:

```go
type Request struct {
    // ...existing fields...

    // Effort controls how thoroughly the model works on the response.
    // Affects thinking depth, response length, and tool call count.
    // Universal across all providers.
    Effort Effort `json:"effort,omitempty"`

    // Thinking controls whether extended/chain-of-thought reasoning is used.
    // This is a mode selector (on/off/auto), not a depth control.
    Thinking ThinkingMode `json:"thinking,omitempty"`

    // (removed) ThinkingEffort
    // (removed) OutputEffort
}
```

### New Types

```go
// Effort controls how hard the model works across all dimensions.
type Effort string

const (
    EffortUnspecified Effort = ""       // provider picks default
    EffortLow        Effort = "low"    // fast, cheap, less thorough
    EffortMedium     Effort = "medium" // balanced
    EffortHigh       Effort = "high"   // thorough, slower
    EffortMax        Effort = "max"    // maximum capability (not all models)
)

// ThinkingMode controls whether the model uses extended thinking.
type ThinkingMode string

const (
    ThinkingAuto ThinkingMode = ""    // provider/model decides (default)
    ThinkingOn   ThinkingMode = "on"  // force thinking on
    ThinkingOff  ThinkingMode = "off" // force thinking off
)
```

### Decisions

1. **`Coding()` preset**: `EffortHigh` + `ThinkingOn`
2. **`newDefaultRequest()` preset**: `EffortLow` + `ThinkingOff`
3. **`EffortMax` on unsupported models**: Silent downgrade to `EffortHigh`. No error.
4. **CLI with no flags**: `EffortUnspecified` + `ThinkingAuto` (zero values pass through to the provider — provider applies its own default)

### Default Hierarchy

There are three entry points with different default expectations:

| Entry point | `Effort` | `Thinking` | Rationale |
|---|---|---|---|
| **CLI (no flags)** | `Unspecified` | `Auto` | Provider decides everything. User gets the model's natural behavior. |
| **`newDefaultRequest()`** | `Low` | `Off` | Conservative programmatic default. Callers opt in to more. |
| **`Coding()` preset** | `High` | `On` | Coding tasks benefit from thorough reasoning. |

When the provider receives `EffortUnspecified`, it applies its own sensible default:

| Provider | Default for `EffortUnspecified` |
|---|---|
| Anthropic ≥ 4.6 | `output_config.effort: "medium"` |
| Anthropic < 4.6 | `budget_tokens: 31999` (generous, like Claude Code) |
| OpenAI | omit `reasoning_effort` (model decides) |
| Bedrock | `budget_tokens: 31999` |

When the provider receives `ThinkingAuto`, it picks the model's natural thinking mode:

| Provider | Default for `ThinkingAuto` |
|---|---|
| Anthropic ≥ 4.6 | `thinking: {type: "adaptive"}` |
| Anthropic < 4.6 | `thinking: {type: "enabled"}` |
| OpenAI | (omit — model decides based on effort) |
| Bedrock | `reasoning_config: {type: "enabled"}` |

### Effort.ToBudget() Distribution

`Effort.ToBudget(low, high)` is used only by providers that need `budget_tokens` (Anthropic < 4.6, Bedrock). `EffortMax` is silently downgraded to `EffortHigh` before `ToBudget` is called, so the function maps 3 effective levels across the range:

| Effort | Budget (range 1024–31999) |
|---|---|
| `Low` | 1024 (= low bound) |
| `Medium` | 16512 (= midpoint) |
| `High` / `Max` (downgraded) | 31999 (= high bound) |

```go
func (e Effort) ToBudget(low, high int) (int, bool) {
    switch e {
    case EffortLow:    return low, true
    case EffortMedium: return low + (high-low)/2, true
    case EffortHigh:   return high, true
    case EffortMax:    return high, true  // same as high for budget
    default:           return 0, false
    }
}
```

### Provider Mapping

#### Anthropic ≥ 4.6 (Sonnet 4.6, Opus 4.6)

| `Thinking` | `Effort` | API params |
|---|---|---|
| auto | unspecified | `thinking: {type: "adaptive"}`, `output_config: {effort: "medium"}` |
| auto | high | `thinking: {type: "adaptive"}`, `output_config: {effort: "high"}` |
| auto | max | `thinking: {type: "adaptive"}`, `output_config: {effort: "max"}` |
| off | unspecified | `thinking: {type: "disabled"}`, `output_config: {effort: "medium"}` |
| off | high | `thinking: {type: "disabled"}`, `output_config: {effort: "high"}` |
| on | unspecified | `thinking: {type: "adaptive"}`, `output_config: {effort: "medium"}` |
| on | high | `thinking: {type: "adaptive"}`, `output_config: {effort: "high"}` |

- `ThinkingOn` and `ThinkingAuto` both produce `adaptive` (adaptive IS the "on" mode for 4.6)
- `budget_tokens` is never sent — depth is controlled exclusively via `output_config.effort`
- `EffortMax` is supported on both Sonnet 4.6 and Opus 4.6
- `output_config.effort` is always set (even when thinking is disabled — it controls all output)

#### Anthropic < 4.6 (Haiku 4.5, Sonnet 4.5, Opus 4.5)

Thinking is controlled via `budget_tokens`. `output_config.effort` is set independently on models that support it (Opus 4.5 only).

| `Thinking` | `Effort` | Model | API params |
|---|---|---|---|
| auto | unspecified | any | `thinking: {type: "enabled", budget_tokens: 31999}` |
| auto | high | any | `thinking: {type: "enabled", budget_tokens: ToBudget("high")}` |
| auto | low | any | `thinking: {type: "enabled", budget_tokens: ToBudget("low")}` |
| off | any | any | `thinking: {type: "disabled"}` |
| off | high | Opus 4.5 | `thinking: {type: "disabled"}`, `output_config: {effort: "high"}` |
| off | high | Haiku 4.5 | `thinking: {type: "disabled"}` (no effort support) |
| on | unspecified | any | `thinking: {type: "enabled", budget_tokens: 31999}` |
| on | high | any | `thinking: {type: "enabled", budget_tokens: ToBudget("high")}` |
| any | max | any | silent downgrade to `high` |

- `Effort.ToBudget(low, high)` maps effort levels to budget_tokens range
- `output_config.effort` is set on Opus 4.5 (which supports it); omitted on Haiku/Sonnet 4.5 — independently of thinking mode

#### OpenAI

`Thinking` mode maps differently per model category since not all models support `reasoning_effort: "none"`.

| `Thinking` | `Effort` | Model category | API params |
|---|---|---|---|
| auto | unspecified | any | (omit reasoning_effort — model decides) |
| auto | high | any | `reasoning_effort: "high"` |
| auto | low | any | `reasoning_effort: "low"` |
| off | any | non-reasoning | (no-op — model can't think) |
| off | any | pre-GPT-5.1 | omit reasoning_effort (can't disable; ignore gracefully) |
| off | any | GPT-5.1+ | `reasoning_effort: "none"` |
| off | any | Codex | `reasoning_effort: "none"` |
| on | unspecified | non-reasoning | (no-op — model can't think) |
| on | unspecified | reasoning | `reasoning_effort: "high"` (default to high) |
| on | high | reasoning | `reasoning_effort: "high"` |
| any | max | Codex | `reasoning_effort: "xhigh"` |
| any | max | other | silent downgrade to `"high"` |

- `ThinkingOff` on pre-GPT-5.1 reasoning models: silently ignored (these models can't disable reasoning)
- `ThinkingOn` on non-reasoning models: silently ignored (these models can't reason)
- `Effort` maps directly to `reasoning_effort` (the values are the same strings)

#### Bedrock

| `Thinking` | `Effort` | API params |
|---|---|---|
| auto | unspecified | `reasoning_config: {type: "enabled", budget_tokens: 31999}` |
| auto | high | `reasoning_config: {type: "enabled", budget_tokens: ToBudget("high")}` |
| off | any | omit reasoning_config |
| on | any | `reasoning_config: {type: "enabled", budget_tokens: ToBudget(effort)}` |
| any | max | silent downgrade to `high` |

- Same pattern as Anthropic < 4.6 (Bedrock uses the Anthropic wire format)

#### OpenRouter

OpenRouter uses the OpenAI Chat Completions wire format with `reasoning.effort`:

| `Thinking` | `Effort` | API params |
|---|---|---|
| auto | unspecified | (omit reasoning) |
| auto | high | `reasoning: {effort: "high"}` |
| off | any | (omit reasoning — same as unspecified) |
| on | unspecified | `reasoning: {effort: "high"}` |
| any | max | silent downgrade to `"high"` |

- `ThinkingMode` maps to presence/absence of the reasoning block
- No model-category awareness (OpenRouter routes to the right backend)

## Migration

### Approach: Clean Break

This is an internal library — all consumers are in this repo. No backward-compat shim needed. Replace all references in one pass.

### Files to Modify

| File | Change |

Add `Effort.IsEmpty()`, `Effort.Valid()`, `ThinkingMode.Valid()`, `ThinkingMode.IsOff()`, `ThinkingMode.IsOn()`. Update `Validate()` to validate `Effort` and `Thinking` instead of old fields. ### CLI Change

Before:
```
llmcli infer --thinking high --effort medium "prompt"
```

After:
```
llmcli infer --effort high --thinking on "prompt"
# or simply:
llmcli infer --effort high "prompt"
# thinking defaults to auto (provider decides), effort defaults to unspecified (provider decides)
```

No flags at all:
```
llmcli infer "Hello"
# Effort: unspecified, Thinking: auto
# → provider applies its own defaults:
#   Anthropic ≥4.6: adaptive thinking, medium effort
#   Anthropic <4.6: enabled thinking, budget 31999
#   OpenAI: no reasoning_effort set, model decides
```

| Old flag | New flag | Notes |
|---|---|---|
| `--thinking none` | `--thinking off` | |
| `--thinking low` | `--effort low` | thinking defaults to auto |
| `--thinking medium` | `--effort medium` | |
| `--thinking high` | `--effort high` | |
| `--thinking xhigh` | `--effort max` | |
| `--effort low` | `--effort low` | unchanged |
| `--effort high` | `--effort high` | unchanged |
| *(no flags)* | *(no flags)* | both fields zero → provider decides |

## Acceptance Criteria

1. `llm.Request` has `Effort` and `Thinking` fields; `ThinkingEffort` and `OutputEffort` are fully removed.
2. All providers map `Effort` + `Thinking` to their native API without hacks or promotion logic.
3. Anthropic ≥ 4.6 never sends `budget_tokens` on adaptive thinking.
4. Anthropic < 4.6 derives `budget_tokens` from `Effort.ToBudget()`.
5. Anthropic sets `output_config.effort` independently of thinking mode on models that support it (Opus 4.5, Sonnet 4.6, Opus 4.6).
6. OpenAI maps `Effort` directly to `reasoning_effort` with minimal translation.
7. OpenAI `ThinkingOff` is gracefully handled per model category (none / omit / no-op).
8. Bedrock uses `Effort.ToBudget()` for `budget_tokens`.
9. `EffortMax` silently downgrades to `EffortHigh` on models that don't support it.
10. `RequestBuilder.Coding()` uses `EffortHigh` + `ThinkingOn`.
11. `newDefaultRequest()` uses `EffortLow` + `ThinkingOff`.
12. CLI with no flags passes `EffortUnspecified` + `ThinkingAuto` — provider decides.
13. `Validate()` validates `Effort` and `Thinking` (not old fields).
14. `llmcli infer -v` shows the new fields in request params and provider params.
15. All existing tests updated and passing.
16. No deprecated `ThinkingEffort` or `OutputEffort` references remain in the codebase.
17. `docs/thinking-and-effort.md` documents the design and per-provider mappings.

## Out of Scope

- Provider-specific `budget_tokens` override (power users who want exact token budgets). Can be added later as a provider option.
- `ThinkingMode` values beyond auto/on/off (e.g. "interleaved"). Provider-specific; can be added to provider options if needed.
- New provider support (Vertex AI, etc.).
