# Plan: Fix stream cancellation by making SSE shutdown coordinated and updating the incorrect Anthropic regression

Reference: revised after self-review of the earlier plan. This plan is based on the current implementations in `provider/anthropic/stream.go`, `internal/sse/lines.go`, `provider/openai/api_completions.go`, `provider/openai/api_responses.go`, `provider/openrouter/openrouter.go`, and the current failing test in `provider/anthropic/stream_error_test.go`.

## Reviewed conclusion

The earlier plan was inconsistent: it tried to preserve the invariant “do not close the body while another goroutine is in `Read(...)`” while also proposing a shared cancellation fix that closes the reader to unblock that exact `Read(...)`.

After reviewing the code paths, the more correct invariant for these providers is:

- the body is an HTTP response body (or behaves like one),
- cancellation must return promptly,
- blocked SSE reads must be unblocked on cancellation,
- the scanner goroutine must not be leaked.

Given the current shared helper design (`ForEachDataLine(...)` starts a scanner goroutine), the correct fix is to make the SSE helper coordinate cancellation by closing the reader and waiting for the scanner goroutine to exit. That means the current Anthropic test is asserting the wrong behavior and should be replaced with a regression that verifies the intended shutdown behavior instead.

## Root cause

`internal/sse.ForEachDataLine(...)` launches a scanner goroutine that may block in `Read(...)`. On `ctx.Done()`, the foreground goroutine returns immediately with `ctx.Err()` without coordinating shutdown of the scanner goroutine. Provider callers currently compensate by deferring `body.Close()` outside the helper.

That creates two problems:

1. **close/read race at the provider layer** — the provider closes the body from one goroutine while the scanner goroutine is blocked in `Read(...)`.
2. **goroutine leak risk in the SSE helper** — if the caller does not close the reader, the scanner goroutine may remain blocked forever.

## Correct acceptance criteria

- The Anthropic cancellation regression is updated to assert the correct behavior.
- Cancellation still returns promptly for `ForEachDataLine(...)` and `ParseStream(...)`.
- `ForEachDataLine(...)` closes the reader on cancellation when the reader supports `io.Closer`.
- `ForEachDataLine(...)` waits for its scanner goroutine to exit before returning on cancellation.
- Redundant provider-level `defer body.Close()` wrappers around `ForEachDataLine(...)` are removed from affected providers.
- Targeted tests for `internal/sse`, `provider/anthropic`, `provider/openai`, and `provider/openrouter` pass.

### Task 3: Refactor `ForEachDataLine(...)` to coordinate cancellation and scanner shutdown

**Files modified**: `internal/sse/lines.go`
**Files created**: none
**Estimated time**: 5 minutes

**Code to write**:

Keep the current public signature:

```go
func ForEachDataLine(ctx context.Context, r io.Reader, fn func(Event) bool) error
```

Do **not** widen it to `io.ReadCloser`; that would be a broader API change than necessary.

Instead:

1. Detect whether `r` also implements `io.Closer`.
2. Start the scanner goroutine as today.
3. On cancellation, if a closer is available, call `Close()` exactly once to unblock `Read(...)`.
4. Wait for the scanner goroutine to finish before returning.
5. Normalize any close-induced scanner error to `ctx.Err()` when cancellation triggered shutdown.

Implementation shape:

```go
func ForEachDataLine(ctx context.Context, r io.Reader, fn func(Event) bool) error {
    lines := make(chan scanResult, 16)
    scannerDone := make(chan struct{})

    var (
        closer    io.Closer
        canClose  bool
        closeOnce sync.Once
    )
    if c, ok := r.(io.Closer); ok {
        closer = c
        canClose = true
    }

    closeReader := func() {
        if !canClose {
            return
        }
        closeOnce.Do(func() {
            _ = closer.Close()
        })
    }

    go func() {
        defer close(scannerDone)
        scanner := bufio.NewScanner(r)
        scanner.Buffer(make([]byte, 0, 64*1024), 1024*1024)
        for scanner.Scan() {
            lines <- scanResult{line: scanner.Text()}
        }
        if err := scanner.Err(); err != nil {
            lines <- scanResult{err: err}
        }
        close(lines)
    }()

    var pendingEvent string
    for {
        select {
        case <-ctx.Done():
            closeReader()
            <-scannerDone
            return ctx.Err()
        case res, ok := <-lines:
            if !ok {
                if ctx.Err() != nil {
                    return ctx.Err()
                }
                return nil
            }
            if res.err != nil {
                if ctx.Err() != nil {
                    return ctx.Err()
                }
                return res.err
            }
            // existing event parsing
        }
    }
}
```

Use the repository’s actual style and exact error handling; the snippet is structural guidance only.
